Core Algorithms Hidden Inside The Spine

Generation And Recursion

  • Generation 1 reads source packets without parentCandidateId.
  • No-delta packets become lineage rejects, not scored candidates.
  • Generation 2 reads fixture-authored packets whose parentCandidateId matches selected gen-1 parents.
  • Caps stop unbounded expansion: max generations, children per parent, and total population.
  • Current gen-2 quality: drifted.

Fitness

novelty = .50 sourceAbsence + .30 substrateDistance + .20 hiddenDependents
grounding = .40 signalStrength + .25 mechanismClarity + .25 falsifiability - .10 riskPenalty
  • Inputs are seed text, candidate text, claims, evidence, substrate, mechanism, and source packet refs.
  • Legacy metric hints can exist, but do not author score totals.
  • Decay is attached as engine-time metadata with half-life by subtype.

Selection And Lens

  • Explore schedule prioritizes novelty with a grounding floor.
  • Proof schedule prioritizes grounding with a novelty floor.
  • Both run over the same scored candidate pool.
  • Selector uses Pareto fronts before directional ranking and decay-adjusted score.
  • Current final contrast: 3 stable, 0 replaced, 0 dropped.
  • Lens result: Capstone demo fit, 3 selected candidates scored.

Engineering Invariants And Failure Surfaces

Kernel core is machine-clean

src/ emits typed process facts. Human summaries live in tools/microscope and out/**.

src/contracts/index.ts, tools/microscope/*.ts

One shared scored pool

Explore and Proof must select from the same ScoredCandidatePool so contrasts mean something.

kernel.selection-input.v2 -> kernel.selection-comparison.v2

Two fitness axes survive selection

Novelty and grounding remain separate; directional score is a selection view, not a new fitness truth.

src/fitness.ts, src/select.ts

Lens is not fitness

Feasibility is emitted as LensResult after selection and never written into FitnessScore.

kernel.lens-input.v1 -> kernel.lens-result.v1

Goal checks are cheap tripwires

TraceEvents carry checks; tools flatten them into RunTrace.goalChecks and build exits nonzero on failures.

src/trace.ts, tools/run.ts

Fixture failures are hard stops

Bad fixtures, unknown operators, incomplete candidate packets, or empty fixture dirs throw before proof output pretends success.

assertSeedFixture, operatorFor, fixturePaths

No-delta is not hidden

Source packets that do not produce candidates stay visible as lineage rejected nodes.

LineageLedger.rejected

Generated artifacts are disposable

out/** is for inspection and replay, not durable architecture truth.
