Architecture Walkthrough

Contracts cross boundaries. Modules transform them.

module TypeScript contract data flow
kernel boundary
boundary contract
interface SeedFixture {
seed: Seed; // FSD unlock consequence map
sourcePackets: SourcePacket[]; // 11 packets
operators: ReproductionOperator[]; // 5 rules
}

This is the same input shown in the top-left card.

into Generate
module

Generate

Build candidate ideas from source packets that contain a real change.

  1. read sourcePackets
  2. use operators to produce candidate deltas
  3. send 9 candidates forward; record 2 no-delta rejects in lineage
emits
boundary contract
interface CandidatePool {
seed: Seed;
candidates: Candidate[]; // 9 kept
lineage: LineageLedger; // 2 rejected
}

No-delta rejects are kept in lineage, not scored as candidates.

into Fitness
module

Fitness

Attach two scores to every generated candidate.

  1. novelty uses source absence, substrate distance, and hidden dependents
  2. grounding uses signal strength, mechanism clarity, falsifiability, minus risk penalty
  3. emit the same pool with scores and short reasons attached
emits
boundary contract
interface ScoredCandidatePool {
seed: Seed;
candidates: ScoredCandidate[]; // fitness added
}

Selection reads this pool; it does not generate new candidates.

into Select
module

Select

Run two selectors over the same scored candidates.

  1. diverge prioritizes novelty with a grounding floor
  2. converge prioritizes grounding with a novelty floor
  3. compare winners: 3 stable, 0 replaced
emits
boundary contract
interface SelectionComparison {
focus: SelectionResult; // diverge
alternate: SelectionResult; // converge
contrasts: DialContrast[]; // 3 stable, 0 replaced
}

The swap summary above is a projection of this contract.

into Lens
module

Lens

Apply capstone-demo-fit after selection without rewriting fitness.

  1. read selected candidates and observer constraints
  2. score feasibility separately from novelty and grounding
  3. emit lens results for trace and microscope tools
emits
boundary contract
interface LensResult {
lensId: string; // capstone-demo-fit
label: string; // Capstone demo fit
scores: LensScore[]; // 3 scored
}

Lens scores do not enter FitnessScore.

into Trace
module

Trace

Package the run so tools can inspect it without rerunning the kernel.

  1. copy lineage, boundary contracts, generation summaries, events, and goal checks
  2. attach the selection comparison and lens results
  3. emit one JSON-safe run record
emits
boundary contract
interface RunTrace {
caps: RunCaps;
generations: GenerationSummary[]; // 2 generation records
runId: string;
lineage: LineageLedger;
events: TraceEvent[];
comparison: SelectionComparison;
lensResults: LensResult[]; // 1 lens result set(s)
}

Human views read this after the kernel finishes.
